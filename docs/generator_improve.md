# Dataset Generator Improvement Plan

This document plans the extraction of a shared dataset generator into:

```text
tests/dataset_generator.py
```

No implementation is included here. The goal is to make every benchmark script use the same workload definition, seed policy, dataset format, and metadata.

## Current State

The strongest shared-dataset logic is currently embedded in `tests/test_compare_basic.py`:

```text
choose_set_sizes()
make_dataset()
write_dataset()
prepare_datasets()
```

Other scripts such as `tests/test_dlk.py`, `tests/test_find_best_m.py`, and `tests/test_spatial.py` duplicate parts of the workload policy, especially:

```text
choose_set_sizes()
seed layout
ca/cb selection
dataset_mode metadata
```

This is workable for early experiments, but it makes later comparisons fragile. A benchmark row can use the unified JSON schema and still be hard to compare if the underlying data was not generated by the same rules.

## Goals

1. Put dataset generation in one reusable module.
2. Keep all algorithms on the exact same Alice/Bob input sets when an experiment is meant to compare algorithms.
3. Preserve the existing dataset file format so current benchmark wrappers keep working.
4. Make workload parameters explicit and serializable.
5. Record enough metadata to reproduce each trial.
6. Support both file-backed datasets and internal-generator experiments.
7. Avoid touching external algorithm subprojects.

## Non-Goals

The first version should not try to model every possible real-world distribution. It should stay conservative and reproduce the current behavior first.

Out of scope for the first implementation:

```text
heavy-tailed values
timestamp distributions
duplicate/multiset workloads
streaming updates
binary dataset format
large on-disk dataset cache eviction
```

These can be added after the generator is stable.

## Dataset Format

Keep the current text format:

```text
# compare-dataset-v1 ca=10000 cb=10000 d=1000 seed=114514 trial=0 trial_seed=114514
A 10000
...
B 10000
...
```

The header should remain easy to parse. The values should remain one integer per line so all existing C++/Go wrappers can consume it.

The generator should also support reading the same format back:

```python
load_dataset(path: Path) -> Dataset
```

The loader is useful for verifier tests and for future plotting/debug scripts.

## Core Types

Use small dataclasses so the generation contract is explicit:

```python
@dataclass(frozen=True)
class DatasetConfig:
    d: int
    ca: int
    cb: int
    seed: int
    value_modulus: int = 2_147_483_647
    min_value: int = 1
    overlap_policy: str = "replace_common_positions"
    shuffle_policy: str = "shuffle_both"
    duplicate_policy: str = "unique"

@dataclass(frozen=True)
class Dataset:
    alice: list[int]
    bob: list[int]
    metadata: dict[str, Any]
```

Keep `value_modulus` compatible with XYZ's field by default. If a later baseline supports a different value domain, the generator can expose that as an explicit parameter.

## Public API

The first implementation should expose:

```python
def choose_set_sizes(d: int, max_set_size: int, scale: int, minimum: int = 1000) -> tuple[int, int]:
    ...

def trial_seed(base_seed: int, trial: int) -> int:
    ...

def dataset_id(config: DatasetConfig, trial: int) -> str:
    ...

def make_dataset(config: DatasetConfig, trial: int = 0) -> Dataset:
    ...

def write_dataset(path: Path, dataset: Dataset) -> None:
    ...

def load_dataset(path: Path) -> Dataset:
    ...

def prepare_datasets(config: DatasetConfig, trials: int, output_dir: Path) -> list[Path]:
    ...
```

`prepare_datasets()` should be the main entry point for `test_compare_basic.py`. It should generate:

```text
<output_dir>/<dataset_id>/trial0.sets
<output_dir>/<dataset_id>/trial1.sets
...
```

## Generation Semantics

The first version should match the current `make_dataset()` behavior:

1. Validate that `d >= abs(ca - cb)`.
2. Validate that `d` and `abs(ca - cb)` have the same parity.
3. Compute:

```text
replacements = (d - abs(ca - cb)) / 2
```

4. Draw unique base values.
5. Build Alice and Bob from the shared prefix.
6. Replace `replacements` positions in Bob with fresh unique values.
7. Shuffle both sets.

This guarantees:

```text
|A| = ca
|B| = cb
|A symmetric_difference B| = d
```

## CLI Interface

The module should also be runnable as a small tool:

```powershell
python tests\dataset_generator.py --d 1000 --ca 10000 --cb 10000 --trials 5 --seed 114514 --output-dir tests\tmp\datasets
```

Recommended CLI options:

```text
--d
--ca
--cb
--trials
--seed
--output-dir
--max-set-size
--set-size-scale
--value-modulus
--min-value
--dry-run
--manifest
```

If `--ca` or `--cb` is omitted, the CLI can use `choose_set_sizes()`.

When `--manifest` is enabled, write:

```text
manifest.json
```

with one row per generated dataset path and its metadata.

## JSON Schema Integration

The generator should produce metadata that maps directly into `benchmark.v1` rows:

```text
dataset_id
dataset_path
dataset_dir
dataset_mode = "shared_file"
d
ca
cb
seed
trial
trial_seed
value_modulus
overlap_policy
shuffle_policy
duplicate_policy
```

`test_compare_basic.py` should keep using shared dataset files and set:

```text
dataset_mode = "shared_file"
```

Scripts that still rely on C++ internal generation should set:

```text
dataset_mode = "internal_generator"
```

Later, threshold-search scripts can be migrated to shared files when their benchmark wrappers support `--dataset`.

## Migration Plan

### Step 1: Add `tests/dataset_generator.py`

Move the current implementation from `test_compare_basic.py` into the shared module, preserving behavior exactly.

Add unit-style smoke tests through the module CLI:

```powershell
python tests\dataset_generator.py --d 100 --ca 1000 --cb 1000 --trials 2 --seed 114514 --output-dir tests\tmp\dataset_generator_smoke
```

Verify:

```text
two files are created
both files can be loaded
set sizes match ca/cb
symmetric difference equals d
same seed regenerates identical files
different trial produces different files
```

### Step 2: Migrate `test_compare_basic.py`

Replace local functions with imports:

```python
from dataset_generator import DatasetConfig, choose_set_sizes, prepare_datasets
```

Keep output paths and dataset file format unchanged.

Expected behavior after migration:

```text
same command line
same benchmark wrappers
same result JSON schema
same shared-file comparison semantics
```

### Step 3: Share Size and Seed Policy

Use `choose_set_sizes()` and a shared seed helper in:

```text
tests/test_dlk.py
tests/test_find_best_m.py
tests/test_spatial.py
tests/test_z.py
```

At this stage, these scripts may still use internal C++ generation, but their `ca`, `cb`, and seed policies should match the common generator.

### Step 4: Optional Shared-File Threshold Runs

If threshold-search benchmarks support `--dataset` in the relevant mode, add an option:

```text
--dataset-mode internal_generator|shared_file
```

Default can remain `internal_generator` until runtime and storage cost are understood.

## Testing Plan

Use three levels of testing.

### Generator Smoke Test

Generate tiny datasets and load them back.

```powershell
python tests\dataset_generator.py --d 10 --ca 100 --cb 100 --trials 3 --seed 1 --output-dir tests\tmp\dataset_generator_smoke
```

Check:

```text
sizes
difference count
metadata
determinism
```

### Compare Smoke Test

Run:

```powershell
python tests\test_compare_basic.py --algorithms xyz_v2,iblt --d-values 100 --trials 2 --capacity-factors 1.3 --output-dir tests\results\dataset_generator_compare_smoke --keep-datasets
```

Then verify JSON:

```powershell
python tests\json_verifier.py tests\results\dataset_generator_compare_smoke\raw.jsonl --strict
```

### Regression Check

For a fixed config, compare old and new generated files before deleting the old code. The first implementation should produce identical files for:

```text
d=100
ca=1000
cb=1000
seed=114514
trial in {0, 1, 2}
```

If exact byte-for-byte equality is not preserved, document why. The preferred first version is byte-for-byte compatible.

## Risks

1. Some wrappers assume values fit a specific field. Keep `value_modulus` default compatible with XYZ.
2. Large shared-file experiments can produce many files. Keep cleanup behavior in `test_compare_basic.py`.
3. Changing seed semantics can invalidate previous results. Preserve current seed layout first.
4. The generated symmetric difference must remain exactly `d`; add validation after generation.

## Completion Criteria

This task is complete when:

```text
tests/dataset_generator.py exists
test_compare_basic.py imports it instead of owning dataset generation
generated dataset files remain compatible with every benchmark wrapper
strict JSON verification still passes
the smoke commands above pass
docs mention how to generate and inspect datasets manually
```
